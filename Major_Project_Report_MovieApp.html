ABSTRACT

The project "S-Show: Movie and TV Discovery Web Application" is a modern web application developed using React, Vite, Redux Toolkit, and Tailwind CSS for browsing and exploring movies, TV shows, and popular people. The system integrates with the TMDB API to fetch real-time entertainment content such as trending media, popular titles, detailed metadata, recommendations, and trailers.

The application is designed around a modular front-end architecture with reusable UI components and Redux-based state management for media details. It supports category-based filtering, search, infinite scrolling for content loading, and trailer playback through YouTube embedding. The application improves user experience by providing responsive navigation, quick media discovery, and consolidated detail views.

This report presents the complete system analysis, methodology, implementation, and evaluation of the project as per academic report standards.

Keywords: React, Vite, Redux Toolkit, TMDB API, Tailwind CSS, Infinite Scroll, Media Discovery

CHAPTER 1: INTRODUCTION

1.1 AIM

To design and develop a responsive and user-friendly web application that enables users to discover trending and popular movies, TV shows, and celebrities, view comprehensive media details, and watch trailers through an integrated interface.

1.2 PROJECT BACKGROUND

Entertainment users increasingly rely on digital platforms for discovering content. Existing large OTT and media platforms often provide rich data but are heavy and complex for quick exploration. The need exists for a lightweight, focused, and fast web app that offers:

  • Trending and category-wise media discovery
  • Unified search across media types
  • Trailer viewing
  • Recommendation-based navigation

This project uses TMDB's public APIs and a modern React stack to provide an efficient entertainment discovery experience.

1.3 OBJECTIVE

  • Build a single-page application for movie and TV discovery.
  • Integrate TMDB API for dynamic data.
  • Implement category and duration filters.
  • Implement infinite scrolling for scalable browsing.
  • Provide detailed pages for movies, TV shows, and people.
  • Display trailers and recommendations for better engagement.
  • Maintain modular, reusable component architecture.

1.4 BENEFITS

  • Fast and lightweight interface for media browsing.
  • Real-time data from a trusted movie database.
  • Better user engagement through trailers and recommendations.
  • Extensible structure for future features (authentication, watchlist, reviews).
  • Practical implementation of modern front-end engineering practices.

CHAPTER 4: PROPOSED METHODOLOGY

4.1 SYSTEM ARCHITECTURE

The application follows a client-side SPA architecture:

  1. User interacts with React UI components.
  2. Components trigger API calls via Axios instance.
  3. For detail pages, data is fetched through Redux async actions.
  4. API responses are rendered in reusable UI modules.

Figure 1: High-Level Architecture

4.2 PHYSICAL DESIGN

  • Platform: Web browser

  • Data source: TMDB cloud API

4.3 DATABASE DESIGN

Design characteristics:

  • Dark theme layout
  • Tailwind utility-first styling
  • Sidebar + top search navigation
  • Card-based responsive visual hierarchy

5.2 MODULES AND THEIR ABSTRACTION

  1. Routing Module (App.jsx)
    Handles URL-to-component mapping for all sections.

  2. State Module (store/Store.jsx + slices)
    Manages detail-state for movie, tv, person.

  3. API Module (utils/Axios.jsx)
    Centralized axios instance with TMDB base URL and auth header.

  4. List Modules (Trending, Popular, Movie, Tv_show, People)
    Fetch and render paginated collection views with filters.

  5. Detail Modules (Moviedetails, Tvdetails, Peopledetails)
    Fetch complete metadata and render rich details.

  6. Search Module (Topnav)
    Provides multi-type search and direct navigation.

  7. Trailer Module (Trailer)
    Renders YouTube trailer via react-player.

5.3 RESULT ANALYSIS

Observed outcomes:

  • Correct loading of trending, popular, and category data from TMDB.
  • Successful list pagination using infinite scroll.
  • Proper route navigation between listing and detail pages.
  • Detail pages show key metadata (overview, ratings, genres, translations).
  • Trailer playback works when trailer key is present.
  • Search provides quick cross-category lookup.
